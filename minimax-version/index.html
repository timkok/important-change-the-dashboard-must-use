    What This Measures

    GDELT baseline

    GDELT is used to collect public web news coverage matching Novo Nordisk, Eli Lilly, and GLP-1 product or molecule terms. It provides article metadata that can support media-exposure monitoring.

    GDELT does not cover every paid database, broadcast clip, social post, paywalled impression, or paid social-listening export. It should be treated as an auditable public-media baseline.

    Metrics

    Plain-English formulas
    • Share of voice = company mentions / total mentions.
    • Weighted SOV uses weightedExposure = 1 + normalizedReach x 0.35 + normalizedEngagement x 0.25 + normalizedSourceAuthority x 0.25 + recencyWeight x 0.15.
    • Sentiment score = (positive mentions - negative mentions) / total mentions.
    • Reach, engagement, and sourceAuthority are proxies for GDELT records. Engagement is 0 unless imported from CSV or paid tools.

    Classification

    Rules, not LLMs

    Topic classification uses keyword rules in the Python normalization pipeline. Standalone Novo/Lilly terms require GLP-1, obesity, diabetes, or pharma context to reduce false positives.

    Deduplication uses normalized URL first, then title + date + sourceDomain when URL is missing.

    Updates and Imports

    Preserve good data

    Scheduled runs are incremental: the pipeline loads existing generated records, fetches only recent GDELT windows, merges, deduplicates, keeps a rolling window, and preserves old data if a new fetch fails.

    Meltwater, Brandwatch, Talkwalker, or similar exports can be added later by dropping CSV files into minimax-version/data/imports/ using the documented columns.
