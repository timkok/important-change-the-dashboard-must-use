Real Data Transparency & Proxy Mappings

Caveats regarding GDELT DOC 2.0

GDELT does not provide native social engagement or reach. To maintain a functional dashboard, the following proxy mappings are applied:

  • Reach Proxy: Injected based on source authority and tier (Tier 1 = 100,000, Finance = 50,000, Trade = 35,000, Other = 10,000).
  • Engagement Score: Defaulted to 0 for GDELT. Active metrics are only populated when importing files from Meltwater, Brandwatch, or Talkwalker via the local CSV loader.
  • Source Authority: Standardized domain-tier rank from 45 (Other) to 95 (Tier 1).

All charts and explorer cards referencing reach or authority metrics display a "Proxy Metric" label for compliance and transparency.

2. Record Count Transparent GDELT queries with post-fetch false positive context filtering on single-word match tokens.

4. Topic Classification Quality Rules-based classification mapping to 16 specialized obesity/cardiology/policy buckets.
5. Alert Usefulness Tracks spikes in safety issues, legal disputes, price wars, and SOV leads, highlighting real-time issues without mock injections.
6. Dashboard Usability Clean responsive CSS styles with CSS grid, CSS variables, glassmorphic layout, and custom light SVG renderings (no massive d3/Chart.js scripts needed).
7. Deployment Reliability One combined GitHub Actions workflow deploying to subfolder paths on GitHub Pages. Direct absolute links omitted for root path compatibility.
